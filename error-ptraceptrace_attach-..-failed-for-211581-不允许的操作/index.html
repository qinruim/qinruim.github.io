原因：

新版的Linux系统加入了 ptrace-scope 机制,该机制的目的是防止用户访问正在执行的进程的内存，但是如jinfo,jmap这些调试类工具本身就是利用ptrace来获取执行进程的内存等信息。

解决方案：

关闭ptrace-scope防护

1
2
3
4
5
6
7

//临时修改
sudo sysctl -w kernel/yama/ptrace_scope=0

//永久修改
sudo vim /etc/sysctl.d/10-ptrace.conf 
	// 1  修改为 0
kernel.yama.ptrace_scope = 0

本次错误记录到此为止。
